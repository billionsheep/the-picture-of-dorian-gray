# 手动测试计划（MVP）

> 目的：不用写自动化测试，也能确保每个里程碑可验收。

## 启动
1. `npm i`
2. `npm run dev`
3. 浏览器打开本地地址，确认画面出现

## M1：热区点击
- 点击不同热区 → 控制台打印不同 id
- 不点击热区 → 无输出

## M2：对话框
- 点击 letter → 弹出文本
- 点击关闭 → 返回场景
- 重复点击 letter → 可再次弹出

## M3：背包
- 点击 letter → 获得物品 “密码：12:30”
- 背包出现该物品
- 点击物品 → 选中高亮
- 再点击 → 取消选中

## M4：使用道具
- 不选物品点击 drawer → 显示 fail 文本（需要密码）
- 选中 “密码：12:30” 点击 drawer → success（抽屉打开 + 获得 hook）
- 抽屉打开后，再点击 portrait → 触发结尾

## M5：结尾
- 触发 end 后：显示结尾字幕
- 输入锁定（或提供“返回标题”按钮）

> 每次提交前至少跑完一次此测试。
