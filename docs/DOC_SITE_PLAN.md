# 文档站点计划（用于最终提交）

> 目标：你把这一套内容做成一个简单网站（或 PDF），即可满足“5–10页文档/数字形式”要求。

建议结构（Markdown 即可）：
- 01_overview.md：项目概念、交付物、怎么玩
- 02_lit_basis.md：原著与版本差异（审查/压抑）如何进入游戏
- 03_design.md：绣湖风格选择、机制列表、UI示意
- 04_level_mvp.md：第一关“画室”谜题闭环分解（线索→动作→结尾）
- 05_process.md：制作过程、工具链（OpenCode/AI 辅助）、迭代记录
- 06_reflection.md：反思与未来扩展
- 07_bibliography.md：参考文献与素材来源说明

实现方式（任选其一）：
- 方案A：仓库内 Markdown + 截图，最后导出 PDF
- 方案B：用静态站生成器（如 VitePress）生成 docs 并一起部署（如果时间够）

> 不要等到最后写文档。每完成一个里程碑就补一小段 + 一张截图。
